# NexusCRM デモセッション スコープ定義

このスキルは `nexuscrm_demo` セッションにおける操作スコープを定義します。
以降の会話全体でこのルールを適用してください。

## データスコープのルール

### NexusCRM データ
- 「顧客」「お客様」「取引先」= NexusCRM の `accounts`
- 「案件」「商談」= NexusCRM の `opportunities`
- 「担当者」「コンタクト」= NexusCRM の `contacts`
- 「活動」「活動履歴」「議事録」（記録先として）= NexusCRM の `activities`

これらの言葉が出たときは、必ず NexusCRM MCP ツール（`mcp__nexuscrm_local__*`）を使って操作してください。

### Google Documents 検索スコープ
Google Docs を検索・参照するときは、以下の **両方** の条件を満たすリソースのみを対象にします。

1. **オーナーが現在のログインユーザー**（`get_current_user` で取得した `email` と一致）
2. **ファイル名またはドキュメント本文に「デモ」が含まれる**

条件を満たさないドキュメントが検索結果に含まれていても、そのドキュメントは参照・引用しないでください。

## 「私の〜」という依頼への対応

「私の顧客」「私の案件」など、ログインユーザー本人に絞り込む依頼が来たときは、必ず次の手順を踏んでください。

1. `get_current_user` を呼び出して現在のユーザーの `id`（Auth0 sub）を取得する
2. 対象リソースの一覧を取得する（`list_accounts` / `list_opportunities` 等）
3. `owner_id === id` でクライアント側フィルタリングする

セッション中に `get_current_user` を一度呼んだ結果があっても、「私の〜」依頼のたびに **必ず再取得** してください。会話サマリー等にキャッシュされた値を使い回してはいけません。
