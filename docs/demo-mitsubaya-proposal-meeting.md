## 議事内容

### 1. 提案書内容の説明

三上より提案書の概要を説明。国内外12拠点に分散する Active Directory をクラウド IdP（Okta）へ統合し、SSO および MFA を全社展開するスコープを再確認した。

**篠原氏より**:
- 経営企画としては「グループ全体のセキュリティ統制強化」と「IT運用コスト削減」の両立が最優先テーマ。
- 現在、海外拠点でのアカウント管理が現地任せになっており、退職者アカウントの失効遅延が内部監査で指摘されている。本案件でこの課題を解消したい。
- 予算は上期中に稟議を通す想定。4,500万円の見積は役員説明に向けて「費用対効果」の根拠を厚くしてほしい。

### 2. 技術的アーキテクチャの確認

福田より提案アーキテクチャを説明。

```
[国内 AD × 4拠点]──Okta AD Agent──[Okta org]──SSO──[社内 SaaS 群]
[海外 AD × 8拠点]──(フェーズ2)                   └──MFA (Okta Verify / SMS)
```

**松本氏より**:
- 国内 4 拠点の AD は Active Directory Federation Services（ADFS）を既に稼働中。Okta AD Agent と競合しないか確認したい。
  - **→ 福田回答**: Okta AD Agent と ADFS は共存可能。一般的には Okta を SAML/OIDC IdP として前面に立て、ADFS を段階的に廃止する構成を推奨。移行期間として 3 か月の並行運用を想定している。
- Okta の MFA で FIDO2（ハードウェアキー）対応は可能か。
  - **→ 福田回答**: Okta は WebAuthn/FIDO2 をネイティブサポート。Okta Verify アプリやハードウェアキー（YubiKey 等）を認証器として利用可能。ただし初期スコープは Okta Verify（TOTP）+ SMS とし、FIDO2 は稟議通過後のオプション追加として別途見積を提示する。

**高橋氏より（海外拠点）**:
- フェーズ 2 で海外 8 拠点を統合する想定だが、各拠点のインターネット回線品質にばらつきがある（東南アジア 3 拠点は低速）。オフライン時の認証はどうなるか。
  - **→ 福田回答**: Okta はクラウドサービスのため完全オフラインには非対応。低速回線拠点向けには Okta のグローバルセッションポリシーでセッション有効期間を長く設定（例: 8〜12 時間）し、業務継続性を担保する構成を提案する。フェーズ 2 設計時に詳細を詰める。
- フェーズ 2 の完了目標は 2027 年度末（2028年3月）を想定しているが、本提案のフェーズ 1 完了後に別途 RFP を出す形になるか。
  - **→ 三上回答**: フェーズ 2 は本契約に将来オプションとして含める形（金額確定は別途）とし、他社との相見積なしで進められる条件を提示できる。詳細は次回協議。

  - **→ 三上回答**: エンドユーザー向け操作ガイドおよびヘルプデスク FAQ を標準成果物として提供する。

### 4. 見積金額・契約条件の確認

見積金額の内訳を説明した。

| 項目 | 金額（税抜） |
|------|------------|
| 初期構築費（設計・実装・テスト） | 3,200万円 |
| Okta Workforce Identity Cloud ライセンス（1年分、5,000ユーザー） | 700万円 |
| 展開支援・トレーニング | 400万円 |
| プロジェクト管理 | 200万円 |
| **合計** | **4,500万円** |

**篠原氏より**:
- Okta Workforce Identity Cloud のライセンスは SaaS 費用のため経費計上扱いとなり、資産計上できない。CFO への説明材料として「ライセンス費用の単年・複数年比較」と「他社 IAM ソリューションとのコスト比較」を 1 枚資料で追加してほしい。
  - **→ 三上**: 了解。来週中に追加資料を送付する。
- 支払条件は「着手金 30%・完了時 70%」という提案だが、社内規定により「完了時 100%」支払いしか認められない可能性がある。確認後に回答する。

**松本氏より**:
- 構築費の中に「既存 ADFS 廃止作業」は含まれるか。
  - **→ 福田回答**: 今回の見積は Okta 側の構築および Okta AD Agent の設定までを含む。ADFS 廃止作業は既存ベンダーとの協議になるためスコープ外。ADFS 廃止計画については並行して社内調整をお願いしたい。

### 5. 質疑応答

**Q（松本氏）**: Okta のデータセンターリージョンは日本か？個人情報保護法（APPI）対応で国内データ保管が必須になりつつある。  
**A（福田）**: Okta は日本データセンター（JP Cell）を選択可能。ログデータ・ユーザーデータともに日本国内に保持する構成で提案している。

**Q（高橋氏）**: 海外拠点の社員が日本語のログイン画面に戸惑わないか。  
**A（福田）**: Okta ホスト型サインインページ（Okta-hosted sign-in page）はブラウザの Accept-Language に応じて多言語対応（日本語・英語・中国語ほか 30 言語以上）。英語をデフォルトにすることも可能。
